# SSA Catalogue — Project Brief for Claude

Bilingual (FR / EN) static product catalogue for **SSA — Solutions Solaires Adaptées**.
Pure HTML / CSS / vanilla JS + Python build scripts. Deploys to **Vercel**.

---

## 1. Project at a glance

| | |
|---|---|
| **Stack** | Static site — HTML / CSS / vanilla JS — no framework, no build step on the host |
| **Data** | `data/products.json` generated by `build_data.py` from `sources/data ssa.xlsx` |
| **PDFs** | Per-product datasheets (FR + EN) via `generate_datasheets.py` (ReportLab) |
| **Specs** | Per-product spec-grid images via `gen_specs.py` (gated off — needs fonts) |
| **Brand** | SSA — orange `#f7a600`, noir `#1d1d1b`, gris `#969696` (charte 2019). La colonne « ref usine » est interne, **jamais affichée**. |
| **i18n** | French / English, header switch persisted in `localStorage` |
| **Deploy** | Vercel, connected to a private GitHub repo (push = auto-deploy) |
| **Local preview** | `python -m http.server 8765` → `http://localhost:8765/` (ou preview_start `catalogue`) |

## 3. Data sources
`sources/data ssa.xlsx`, feuille **Feuil1**. Colonnes :
`code ssa | ref usine | Designation | Nom produit | Puissance | Flux | Temp Lum | Panel | Batterie`.
Les lignes dont la colonne A ne matche pas `NN NN NN` sont ignorées (titres de
familles, lignes vides). **`ref usine` est une clé interne — jamais rendue.**

## 4. Source → image mapping (CRITICAL)
- Un produit = un préfixe « code ssa » à 5 caractères (`42 00`, `53 01`…),
  déclaré dans `PRODUCTS` (build_data.py) avec ses descriptions/features.
- `assets/images/P2/<code>.png` = photo principale (ex. `42 00 00.png` → Solar Street).
- `assets/images/<code dossier>/` = galerie multi-vues du même produit
  (ex. `50 04 00/` = les 5 montages de l'Azur).
- `build_data.py` **convertit** tout en **WebP** `assets/images/<slug>.webp`,
  `<slug>-2.webp`… (downscale 1280px, q82, `save_webp()`) — le site ne sert **que**
  ces copies normalisées (pas d'espaces dans les URLs). Les PNG originaux (P2/ +
  dossiers galerie) restent en sources pour rebuild (git-suivis, `.vercelignore`d).
- UFO/UFI (64 xx) partagent des codes → distingués par `ref_filter` (UFS01/UFP23).
- **Hard rule : les refs usine (SSP01, ASL23, AGL12…) n'apparaissent nulle part.**
- Produits sans photo dédiée (Solar Street Pro 42 01 04, UFO, UFI) → placeholder
  front-end stylé affichant le code SSA (`code_prefix` = 1er code du produit).

## 5. `products.json` schema
Generated only. Top-level keys: `site`, `families`, `suppliers` (kept empty),
`products`, `table_schemas` (`ssa` = code/designation/power/lumen/efficacy/cct/
panel/battery/dimensions/weight), `i18n` (inc. `tech_labels` + `technical_specs`).
Per product: `id`, `family_id`, `name_slx`, `image`, `gallery[]`,
`description_fr/en`, `features_fr/en`, `datasheet_fr/en`, `table_columns`
(colonnes non vides pour CE produit), `tech_specs[]` (label/valeur du catalogue),
`variants[]`. `efficacy` est calculée (Flux ÷ Puissance). See `build_data.py`.

**Caractéristiques techniques** : les blocs `TECH_SPECS` (matériau, IP/IK, driver,
durée de vie, angle, etc.) et `DIMENSIONS`/`WEIGHTS` (par puissance) de
`build_data.py` proviennent du **catalogue `Catalogue_SSA_2026_light.pdf`**.
⚠️ Les codes/puissances du catalogue ne coïncident PAS toujours avec l'Excel
(ex. Street Pro catalogue = 53 00 00 jusqu'à 80W ; Excel 53 xx jusqu'à 200W ;
Street Core catalogue = 53 00 50). Les dimensions par puissance ne sont mises
au tableau que pour les produits où les W coïncident exactement (Street Plus,
Rock, Aréna, Stadium/Ride, Focus, TUB) ; ailleurs → spec `dimensions` globale.
Rendu : grille `.spec-grid` sur la fiche (script.js) + table 4-colonnes dans le PDF.

## 6. Brand identity (charte graphique SSA 2019)
- **Palette** — orange `#f7a600` (C0 M40 J100 N0), noir `#1d1d1b`, gris `#969696`.
  CSS vars in `:root` (style.css).
- **Typography** — titres **Bebas Neue** (Google Fonts), corps Century Gothic
  → fallback **Jost**. PDF : `assets/fonts/BebasNeue-Regular.ttf` (titres),
  Helvetica (corps).
- **Logo** — `assets/brand/ssa-logo{,-sm,-white,-white-sm}.png`, extraits de la
  charte PDF. Fond transparent.
- **Hard display rules** — jamais de refs usine ni de noms de fournisseurs ;
  pas de déformation/recoloration du logo (interdit par la charte).

## 7. Pages & UX
`index.html` (hero + featured + family cards + featured grid) ·
`catalogue.html` (search + family filter, `?family=` / `?q=` in URL) ·
`product.html?id=<slug>` (mosaïque galerie + chips + tableau des références +
fiche PDF + export CSV + comparateur + prev/next) · `compare.html` (max 4).

## 8. PDF datasheets
A4 portrait, bande orange + logo SSA blanc, image + galerie + description +
badges + tableau des références + footer. `CONTACT = {...}` en tête de
`generate_datasheets.py` est **à compléter** (site, email, téléphone, adresse —
les champs vides sont masqués). Run with the file viewer closed (locked-file
fallback writes `.pdf.new`).

## 9. How to add a family / product
1. Add rows to `sources/data ssa.xlsx` (code ssa au format `NN NN NN`).
2. Drop la photo principale dans `assets/images/P2/<prefix> 00.png` et les
   vues galerie dans `assets/images/<prefix> 00/`.
3. Add a `PRODUCTS` entry in `build_data.py` (descriptions FR/EN + features)
   — et une entrée `FAMILIES` si nouvelle famille.
4. `python -X utf8 build_data.py` puis `python -X utf8 generate_datasheets.py`.
5. `git add -A && git commit -m "..." && git push` → Vercel auto-deploys.

## 12. Conventions & gotchas
- Never edit `data/products.json` by hand — regenerate.
- Use `python -X utf8` on Windows (encoding).
- Quote paths with spaces in PowerShell ; **PowerShell `-replace` est
  insensible à la casse** (utiliser `-creplace`).
- Feature chips propres et spécifiques (IP66, IK08, `200 lm/W`, `2300-6500K`,
  plages `50-200W`) — pas d'inventions : uniquement les données du fichier Excel
  ou du catalogue PDF SSA.
- Local server : utiliser `ThreadingHTTPServer` (le `TCPServer` single-thread
  se bloque sur les connexions keep-alive du navigateur).
- Keep UI changes reversible; back up before restyling.
